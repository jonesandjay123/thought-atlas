# Firestore Schema Draft

Status: draft for dry-run export only. Do not write to Firebase until the sync step is explicitly enabled.

Firebase project:

```text
project_id: thought-atlas
firestore: Production mode, nam7
hosting: enabled
```

## Design goals

- Firestore is a downstream mirror of local Thought Atlas state.
- Local files remain the source of truth during the current phase.
- Sync should be deterministic and idempotent: the same local state produces the same document ids.
- UI should be able to render graph/search/report views without reading GitHub files directly.
- Firebase must not participate in extraction, reasoning, or graph patch decisions.

## Collection layout

```text
thoughtAtlasMeta/current
thoughtSources/{sourceId}
thoughtDigests/{digestId}
thoughtNodes/{nodeId}
thoughtEdges/{edgeId}
thoughtReports/{sourceId}
thoughtRegistryRuns/{runId}
```

## `thoughtAtlasMeta/current`

One document describing the exported snapshot.

Fields:

```text
schema_version: string
project_id: string
exported_at: timestamp-string
source_count: number
digest_count: number
node_count: number
edge_count: number
report_count: number
registry_run_count: number
local_graph_updated_at: timestamp-string | null
```

## `thoughtSources/{sourceId}`

Mirrors source manifests and registry source state.

Fields:

```text
source_id: string
title: string
source_type: string
status: string
path: string
manifest_path: string
content_hash: string
tags: string[]
origin: map | null
first_seen_at: timestamp-string | null
last_seen_at: timestamp-string | null
created_at: timestamp-string | null
updated_at: timestamp-string | null
```

Notes:

- `content_hash` is used for duplicate detection.
- `path` points to local repo path, not Firebase Storage.
- Later, source raw text may either stay local/GitHub-only or be exported into Storage; not part of this draft.

## `thoughtDigests/{digestId}`

Mirrors digest files.

Fields:

```text
digest_id: string
source_id: string
created_at: timestamp-string
summary: string
item_count: number
items: map[]
path: string
```

Notes:

- Store full `items` array for UI inspection.
- If documents get too large later, split into `thoughtDigestItems/{itemId}`.

## `thoughtNodes/{nodeId}`

Mirrors graph nodes.

Fields:

```text
id: string
kind: string
title: string
body: string
source_refs: map[]
confidence: number
tags: string[]
source_ids: string[]
updated_at: timestamp-string | null
```

Notes:

- `source_ids` is denormalized from `source_refs` for filtering.
- Node id is the Firestore document id.

## `thoughtEdges/{edgeId}`

Mirrors graph edges.

Fields:

```text
id: string
from: string
to: string
relation: string
weight: number | null
confidence: number
source_refs: map[]
source_ids: string[]
rationale: string
updated_at: timestamp-string | null
```

Notes:

- UI can join `from` / `to` with `thoughtNodes` client-side.
- `source_ids` supports filtering edges by source.

## `thoughtReports/{sourceId}`

Generated ingest report text and summary.

Fields:

```text
source_id: string
path: string
markdown: string
digest_id: string | null
patch_id: string | null
digest_items: number
patch_operations: number
graph_nodes: number
graph_edges: number
updated_at: timestamp-string | null
```

Notes:

- Reports must be generated by `generate-ingest-report.mjs` / `finalize-ingest.mjs`.
- Do not manually edit report counts.

## `thoughtRegistryRuns/{runId}`

Mirrors registry run log.

Fields:

```text
run_id: string
type: string
source_id: string | null
at: timestamp-string
content_hash: string | null
manifest_path: string | null
source_path: string | null
digest_id: string | null
patch_id: string | null
graph_path: string | null
status: string | null
summary: map | null
origin: map | null
```

## Export strategy

Initial sync should be replace/upsert by deterministic document id:

```text
sources: source_id
digests: digest_id
nodes: node.id
edges: edge.id
reports: source_id
runs: run_id
meta: current
```

No deletes in the first writer unless explicitly requested. A future sync can support deletion by comparing exported ids to remote ids.

## Security stance

Initial Firestore rules should be private/read-protected until UI auth is designed.

Draft rule posture:

```text
allow read, write: if false;
```

or, for authenticated owner-only UI later:

```text
allow read: if request.auth != null && request.auth.token.email in allowedEmails;
allow write: if false; // writes only via service account / Admin SDK
```
