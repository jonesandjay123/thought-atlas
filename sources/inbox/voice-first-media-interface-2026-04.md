# Voice-First Media Agent — 影音內容的對話式知識介面

> ⚠️ **這是發想階段（Early Exploration）**
>
> 這個資料夾不是實作計畫，而是思考起點。目前所有描述都是探索性的，
> 方向可能大幅變動，甚至整個轉向。請不要把這裡的任何內容當成 spec 或承諾。

---

## 這在挑戰什麼？

影音內容現在有一個根本限制：**它只能被「播放」，不能被「操作」**。

你想找某集 podcast 的某段話，你得手動翻。你想理解一支 40 分鐘的 YouTube，你得從頭看。你曾經聽過一個很有感的觀點，但完全不記得在哪裡，就消失了。

這個發想在問的是：

> 能不能把影音內容從「時間軸」解放出來？
> 把只能被播放的東西，變成可以被查、被問、被操作的知識介面？

這不是「做個更好的播放器」，而是換一個接觸內容的方式。

---

## 為什麼值得想？

幾個很常遇到、但目前幾乎無解的情境：

- 想找「哪一集 podcast 有講到某個主題或某個人」，但除了憑記憶沒有別的辦法
- 想快速掌握一支 30 分鐘以上的訪談，不想從頭到尾播
- 曾經聽過某個很對的觀點，但忘記在哪一集、哪個頻道
- 視障使用者想使用影音內容，但完全沒有 timeline UI 的視覺輔助
- 通勤、多工的時候想「用問的」而不是「用找的」

這些不是邊緣需求。它們現在幾乎都沒有好的解法。

---

## 五個可以拆的方向維度

這不是要選一個，而是幫助討論時有座標。

**1️⃣ Retrieval（找得到）**
最直覺的一層——找哪一集講某主題、某句話出現在哪。影音版 Ctrl+F。
可以再分：keyword search vs. 語意 search；片段級 vs. 整支影片級。

**2️⃣ Compression（變短 / 變懂）**
長 podcast → 摘要；訪談 → 重點整理。
關鍵分叉：是「一次性摘要」還是「可互動理解」？這會直接影響產品感。

**3️⃣ Interaction（可以問）**
開始有 AI 感的一層——針對你問的問題回答，而不只是把內容原文丟回來。
「他對這個議題的立場是什麼？」「這段在講什麼？」

**4️⃣ Navigation（用問的在影片裡移動）**
用語言取代 timeline UI。「跳到他開始講 inflation 的地方」「剛剛那段再播一次」。
這個沒人真的做好過，但潛力很大。

**5️⃣ Mode shift（從「看」→「聽 / 說」）**
不是 UI feature，而是使用模式的改變。
如果完全沒有畫面，這個系統還成立嗎？如果可以，就是一個強差異化。

## 還沒想清楚的五題

這些問題目前都還開著，不要假裝已經有答案。

1. 核心價值到底在哪——是搜尋？摘要？還是語音互動？
2. 資料來源怎麼定義——只做自己關注的頻道，還是通用工具？
3. 互動方式要多重——CLI？Web？純語音？
4. 視障族群是這個方向的「起點」，還是「延伸應用」？
5. 使用頻率場景是什麼——偶爾查，還是每天用？沒有高頻場景很難成立。
